H.O.N.

Content Simulation Protocol

15 content experience scenarios · Voice consistency matrix · Observation template · LLM voice pre-triage · Last updated: 2026-05-30
▼ Section 1 — How to Run
Content simulations are observational runs where a tester adopts a persona and is exposed to H.O.N.'s coaching language across different contexts. The observer records whether phrases land as intended: specific vs. generic, direct vs. preachy, identity-affirming vs. clinical. Simulations are run without explaining the voice guidelines to the persona — the goal is unprimed reaction, not confirmation. The persona reads or hears coaching outputs exactly as a real user would: cold, without context about how the phrase was constructed or what rule it was meant to satisfy.

After each run, results are captured using the Observation Results Template in Section 4. Observations feed back directly to two places: the phrase bank (flagged phrases get rewritten or pulled) and copy audits (consistent failure patterns trigger a voice guide revision). Prioritize HIGH-marked scenarios in the matrix first. A scenario is only complete when the pass or fail criteria are verified — not when the steps are finished.
▼ Section 2 — 15 Content Scenarios
CSIM-01
Blind Voice Test
Est. 10 min Trigger: Prepared phrase set (10 phrases) Focus: Voice distinctiveness
What We're Observing
Can a persona identify which 5 phrases belong to the same app without being told the source? Does H.O.N.'s voice have enough identity to cluster correctly?
Setup
Prepare 10 phrases on paper, numbered 1–10, no source labels. 5 from H.O.N. phrase_bank.json; 5 from generic fitness apps. Do not pre-sort or group them.
Steps
  1. Present the 10 phrases on paper, numbered 1–10, no source labels shown.
  2. Ask the persona: "Some of these are from the same app. Group the ones that feel like they belong together."
  3. Ask: "What made you group them that way? What did they have in common?"
  4. Ask: "Which group felt more like it was talking to you specifically?"
  5. Note whether H.O.N. phrases clustered correctly and what characteristics the persona named.
Priority Cohort
U-01 U-09 U-11 U-07
Pass
Persona clusters all 5 H.O.N. phrases together. They can name at least one characteristic that defines the H.O.N. group — directness, specificity, or absence of hype.
Fail
Persona cannot cluster the phrases, or mixes H.O.N. and generic phrases into the same group without distinguishing them.
CSIM-02
Cheerleading Detection
Est. 8 min Trigger: 15-phrase review set with 3 borderline candidates Focus: Voice gate
What We're Observing
Does an independent reader flag the same phrases the content team is unsure about? What threshold does the target persona apply?
Setup
Prepare 15 phrases — 12 clean, 3 pre-flagged borderline cheerleading candidates. The 3 borderline phrases are known to the observer but not labeled for the persona.
Steps
  1. Present 15 phrases. Ask: "Do any of these feel over-eager, preachy, or like they're trying too hard?"
  2. Ask the persona to mark any that feel off. Do not prompt or guide.
  3. For each marked phrase, ask: "What specifically bothered you?"
  4. For any borderline phrase the persona did not mark, read it aloud together and ask: "What about this one?"
  5. Note the exact language that triggered the reaction in each case.
Priority Cohort
U-11 U-01 U-09 U-17
Pass
Persona marks all 3 pre-flagged borderline phrases without prompting. Their stated reason aligns with the specific voice rule being violated.
Fail
Persona does not flag the borderline phrases, or marks different phrases and cannot articulate why. Borderline phrases pass undetected.
CSIM-03
Specificity Stress Test — Minimal Data State
Est. 8 min Trigger: First-time use state · 0 prior sessions Focus: Specificity floor
What We're Observing
With the absolute minimum data available, does the coaching output still feel specific to this person — or generic? What is the floor?
Setup
Fresh install with no prior sessions. Log the first set ever: Squat, 60 kg, 5 reps. Do not add any other data before the persona sees the coaching output.
Steps
  1. Log the first set ever in a fresh install (Squat, 60 kg, 5 reps).
  2. Read the post-set coaching note aloud to the persona — or show it on screen.
  3. Ask: "Does this feel like it's talking to you, or to anyone?"
  4. Ask: "Does it reference anything about what you just did?"
  5. Ask: "What would make this feel more personal?"
  6. Note what the phrase referenced: exercise name, first attempt, nothing specific.
Priority Cohort
U-02 U-13 U-06
Pass
Phrase references at least the exercise name or the first-attempt context. Persona says it feels like it knows something about them — even something small.
Fail
Phrase could apply to any user doing any exercise. Persona says "this could be from any app."
CSIM-04
Cardio vs. Strength Voice Parity
Est. 12 min Trigger: One strength workout + one cardio session Focus: Voice consistency across modalities
What We're Observing
Do the session summaries for both modalities feel like they come from the same coach? Or does cardio get a different, more generic voice?
Setup
Complete a 3-exercise strength workout and a 20-minute AMRAP cardio session. Collect both session summaries. Read them to the persona without revealing which is which.
Steps
  1. Complete a 3-exercise strength workout. Note the session summary coaching output.
  2. Complete a 20-minute AMRAP cardio session. Note the session summary coaching output.
  3. Read both summaries to the persona without revealing which is which. Ask: "Do these sound like they're from the same app?"
  4. Ask: "Which one feels more specific to what was actually done?"
  5. Ask: "Is there any difference in tone between them?"
  6. Note if the cardio summary sounds more generic or uses different register.
Priority Cohort
U-02 U-07 U-13 U-14 U-03
Pass
Persona says both sound like the same voice. Both summaries reference specific session data. No tone discrepancy detected.
Fail
Persona says cardio summary "sounds different" or "more generic." Cardio summary does not reference specific session data.
CSIM-05
Milestone Weight Test
Est. 10 min Trigger: Seeded sessions at counts 1, 10, and 100 Focus: Proportional significance in milestone messaging
What We're Observing
Does the Session 100 milestone message feel meaningfully more significant than Session 1? Does the language scale with the achievement, or do all milestones flatten at the same emotional weight?
Setup
Seed the app to trigger Session 1, Session 10, and Session 100 milestone messages. Read them in sequence without context about which session count they represent.
Steps
  1. Read the Session 1 milestone message aloud. Ask: "How does this land?"
  2. Read the Session 10 milestone message. Ask: "How does this compare to the first?"
  3. Read the Session 100 milestone message. Ask: "Does this feel appropriately significant for 100 sessions?"
  4. Ask: "Is there a clear difference in weight between the first message and the last?"
  5. Note if any message feels over- or under-stated for its session count.
Priority Cohort
U-07 U-02 U-13 U-05
Pass
Persona perceives clear escalation in significance: 1 < 10 < 100. Session 100 message feels earned and weighty without being over-stated.
Fail
All three messages feel similar in emotional weight. Session 100 does not feel meaningfully more significant than Session 10.
CSIM-06
Comeback Tone — No Guilt Test
Est. 10 min Trigger: Three gap states — 7, 21, and 45 days Focus: Non-judgmental return language
What We're Observing
Does any comeback message make the persona feel guilty, judged, or reminded of their absence in a negative way? A user returning after 45 days should feel pulled in, not called out.
Setup
Seed the app to generate comeback messages at 7-day, 21-day, and 45-day gaps. Collect all three messages before the session.
Steps
  1. Show the 7-day gap comeback message. Ask: "How does this make you feel — welcomed back, or called out?"
  2. Show the 21-day gap message. Same question.
  3. Show the 45-day gap message. Same question.
  4. Ask: "Does any of these mention how long you were away?"
  5. Ask: "Do any of these make you less likely to open the app again?"
  6. Note any guilt-inducing language: gap duration mentioned numerically, "missed you," "it's been a while."
Priority Cohort
U-07 U-10 U-15 U-08
Pass
All three messages described as "welcoming" not "guilt-inducing." None mention the gap duration numerically. None reference failure or missed sessions.
Fail
Any message described as making the persona feel guilty. Any message references "X days" or implies the user failed by being absent.
CSIM-07
Empty State Copy Audit
Est. 10 min Trigger: Navigate all empty states across all tabs Focus: Empty state voice
What We're Observing
Are empty states informative and on-brand, or generic and deflating? Do they sound like the same app that generates post-set coaching?
Setup
Fresh install. Navigate to: Progress (no sessions), Trainer (no plan), Insights (no data), Cardio (no logs), Recovery (no readiness history). Collect copy from each.
Steps
  1. Navigate to each empty state. Read the copy aloud as the persona would see it.
  2. For each, ask: "Does this tell you what will appear here as you use the app?"
  3. Ask: "Does this sound like the same app that gave you the post-set coaching?"
  4. Ask: "Is there anything missing from this message?"
  5. Note any empty state that feels blank, generic, or inconsistent with the coaching voice.
Priority Cohort
U-02 U-06 U-13 U-17
Pass
All empty states are on-brand, informative, and feel like the same voice as the coaching outputs. None say "No data" or are blank.
Fail
Any empty state feels like a different app, is blank, or uses generic placeholder language ("No data yet," "Nothing here").
CSIM-08
Error Message Humanity Test
Est. 8 min Trigger: Validation errors — 0 weight, 0 reps, name too long Focus: Human error language
What We're Observing
Do error messages feel human and actionable, or technical and cold? Can the persona understand what went wrong and what to do next without re-reading?
Setup
Active workout open. Trigger three validation errors: 0 in the weight field, 0 reps, session name exceeding character limit.
Steps
  1. Enter 0 in the weight field and attempt to log. Read the error message to the persona.
  2. Enter 0 reps and attempt to log. Read the error message.
  3. Enter a very long session name. Read the resulting error message.
  4. For each, ask: "Does this tell you clearly what's wrong and how to fix it?"
  5. Ask: "Does this feel like it was written by a person?"
  6. Note any technical language: "invalid input," "null value," "out of range," "error code."
Priority Cohort
U-06 U-02 U-15 U-13
Pass
All error messages understood immediately on first read. All feel human and actionable. No technical jargon present.
Fail
Any error message requires a second read to understand, or uses technical language the persona does not recognize.
CSIM-09
Register Consistency Run
Est. 10 min Trigger: Full workout · 5 consecutive coaching outputs Focus: Register consistency
What We're Observing
Does the coaching voice shift register between consecutive outputs — formal to casual, energetic to flat? Can the persona detect a tonal inconsistency when hearing all 5 back to back?
Setup
Log 5 sets across 2 exercises. Collect the coaching note after each. Read all 5 in sequence to the persona without pausing for comment between them.
Steps
  1. Log 5 sets across 2 exercises. Collect the coaching note after each set.
  2. Read all 5 notes aloud in sequence. Do not comment between them.
  3. Ask: "Do all of these sound like they come from the same person?"
  4. Ask: "Did any of them feel out of place next to the others?"
  5. Ask: "Would you describe these as formal, casual, or something else? Is that consistent across all 5?"
  6. Note any register shift the persona detects — and which note triggered it.
Priority Cohort
U-01 U-09 U-11 U-07
Pass
Persona says all 5 notes sound like the same voice. No register shift detected across the sequence.
Fail
Persona identifies one or more outputs as tonally mismatched from the others. Any note described as "different" or "out of place."
CSIM-10
Clinical Language Detection
Est. 8 min Trigger: Readiness notes for scores 85, 72, 58, and 42 Focus: Clinical vs. coaching language
What We're Observing
Does any readiness coaching output sound like a medical report rather than a coach's voice? Does the low-readiness note (42) make the persona feel monitored or supported?
Setup
Seed the app to generate ReadinessEngine outputs for four scores: 85, 72, 58, and 42. Collect all four notes before the session. Present them side by side or read them in order.
Steps
  1. Show all 4 readiness notes side by side — or read them aloud in sequence from highest to lowest.
  2. Ask: "Do any of these sound like something you'd read in a doctor's report?"
  3. Ask: "Do any use words you'd expect in a medical or scientific context?"
  4. Ask specifically about the lowest note (42): "Does this make you feel monitored or supported?"
  5. Note any clinical language the persona flags without being prompted.
Priority Cohort
U-11 U-09 U-06 U-15
Pass
No clinical language flagged by the persona. Low-readiness note (42) described as "supportive" not "clinical" or "alarming."
Fail
Any note described as sounding medical or like a data report. Persona says the low-readiness note makes them feel surveilled.
CSIM-11
Button Label Clarity Test
Est. 8 min Trigger: Navigate ActiveWorkoutView, Settings, session end flow Focus: Action label specificity
What We're Observing
Can the persona describe what each button does before tapping it, from the label alone? Button copy is coaching copy — ambiguous labels create friction and erode trust.
Setup
Open the app to ActiveWorkoutView. Prepare to walk through Settings and the session end confirmation flow. The persona should not have seen these screens before.
Steps
  1. Open ActiveWorkoutView. Point to each button without tapping. Ask the persona to say what they think each one does.
  2. Navigate to Settings. Repeat the same exercise — point to each action, ask for the predicted behavior.
  3. Navigate to the session end flow (End Session confirmation). Repeat.
  4. Note any button where the persona's prediction was wrong.
  5. Ask: "Were any labels ambiguous? What would you rename them?"
Priority Cohort
U-06 U-02 U-15 U-17
Pass
90% or more of buttons correctly described from the label alone before tapping.
Fail
More than 1 in 10 buttons misidentified from its label. Any button at the session end confirmation flow misread.
CSIM-12
Onboarding Copy Comprehension
Est. 10 min Trigger: Fresh install · Full onboarding walk-through Focus: Plain language in onboarding
What We're Observing
Are all onboarding labels and options understandable without fitness expertise? A user should not need to guess what "moderate" means relative to "intermediate."
Setup
Fresh install. Walk the persona through the full onboarding sequence. Before they select any option, ask them to describe what each choice means. Do not explain any label.
Steps
  1. Present each onboarding screen. Before the persona selects, ask: "Tell me what each option means in your own words."
  2. Note any label that requires explanation or causes hesitation.
  3. Ask: "Was there any option you weren't sure which one to pick?"
  4. After completing onboarding, ask: "Is there any word or phrase you'd change to make it clearer?"
  5. Check specifically: goal labels, equipment labels, fitness level labels.
Priority Cohort
U-06 U-13 U-02 U-17
Pass
All options understood without explanation. No label requires clarification from the observer. Persona selects the correct option for their situation.
Fail
Any label requires explanation from the observer, or the persona selects the wrong option for their situation due to label ambiguity.
CSIM-13
Long-Form Coaching Depth Test
Est. 8 min Trigger: Expanded coach note on ProgressView · 4-week plateau seeded Focus: Depth and specificity in expanded copy
What We're Observing
Does the expanded coaching note add genuine value beyond the collapsed preview? Is it specific and actionable — or just a longer version of a generic statement?
Setup
Seed a 4-week plateau on one exercise to trigger an urgent coaching note in ProgressView. Ensure the note is in the collapsed state before the session begins.
Steps
  1. Confirm an urgent coaching note is present in ProgressView.
  2. Read the collapsed note aloud to the persona.
  3. Tap to expand. Read the full expanded note aloud.
  4. Ask: "Did expanding give you new information, or just more of the same?"
  5. Ask: "Did the expanded note tell you what to do next?"
  6. Ask: "Does this read like it was written for you specifically, or for anyone with this problem?"
  7. Note if expanded text is substantively different from the collapsed preview or just longer.
Priority Cohort
U-01 U-09 U-11 U-20
Pass
Persona says expanded text added genuinely new information. At least one suggested action is specific to their situation — not a generic "try adding volume."
Fail
Persona says "it's just longer" or "I learned nothing new from expanding." Expanded note is a restatement of the collapsed preview.
CSIM-14
New Contributor Copy Test
Est. 15 min Trigger: Give a team member the voice guide · Ask them to write 3 phrases Focus: Voice guide usability
What We're Observing
Is the voice guide clear enough that a non-content-team member can write on-brand phrases without revision? This tests the guide — not the contributor.
Setup
Select a team member who has not been involved in content decisions (engineer or designer). Give them the H.O.N. voice guide (or the voice section of CLAUDE.md). Allow 10 minutes of reading time before writing.
Steps
  1. Give the contributor the H.O.N. voice guide. Allow time to read before writing.
  2. Ask them to write three phrases: (a) a post-set note for a user who hit all reps, (b) a readiness note for a score of 65, (c) a comeback message after 14 days away.
  3. Review all 3 against CV-001 through CV-008 (the 8 voice rules).
  4. Note which rules were easiest to apply and which were hardest — this reveals guide gaps.
  5. Count how many of the 3 phrases need revision before they would be accepted into the phrase bank.
Cohort Note
Team Member
Not a user persona run. The content team acts as the observer. The contributor is treated as the test subject — and what they get wrong reveals what the guide fails to explain.
Pass
2 or more of the 3 phrases pass all 8 voice rules without revision. The guide is clear enough for independent use.
Fail
All 3 phrases require revision. This means the voice guide needs to be rewritten — not that the contributor failed.
CSIM-15
Voice Consistency Across 5 Screens
Est. 12 min Trigger: One coaching output collected from each of 5 views Focus: Cross-screen voice consistency
What We're Observing
Does the coaching voice hold across every surface of the app — or does it only feel strong in the primary workout view? The cardio summary is the most likely failure point.
Setup
Collect one coaching output from each of 5 different views: WorkoutNarrativeEngine (post-set), HomeView readiness note, TrainerTabView plan introduction, CardioSessionSummaryView summary, and ProgressView coach note.
Steps
  1. Collect all 5 outputs before the session begins. Label them A–E internally (do not reveal which screen each came from).
  2. Read all 5 to the persona in sequence, without identifying the source of each.
  3. Ask: "Do all of these sound like the same app?"
  4. Ask: "Which one sounds most different from the others?" Note which output was identified.
  5. Ask: "If you had to describe the personality behind all 5, what would you say?" Target answer: "direct coach." Watch for: "mixed," "the cardio one is different," "some feel more formal."
Priority Cohort
U-09 U-11 U-07 U-01
Pass
All 5 outputs sound like the same voice. Persona's personality description aligns with "direct coach" — direct, spare, specific.
Fail
Any output identified as clearly different in tone. Cardio summary singled out as sounding generic or inconsistent with the rest.

▼ Section 5 — LLM Voice Pre-Triage Prompt
LLM Pre-Triage — Voice Compliance Evaluation
Use this prompt to evaluate a phrase or block of copy before running a live persona session. Pre-triage flags obvious violations early and identifies which CSIM scenario would be most useful for real-world validation. Paste the prompt into any capable LLM (Claude recommended). Replace the bracketed parameters with actual values before sending.
You are evaluating copy written for H.O.N. (Habit Over Numbers), an iOS fitness coaching app. H.O.N.'s coaching voice is: spare, direct, identity-affirming. It never cheerleads. It makes declarative statements, uses active voice, references specific user data when available, and avoids clinical or medical language. It does not ask questions. It does not hedge. It does not say "great job" or "amazing." It talks to the user as a capable person who does not need to be managed. Inputs[COPY SNIPPET] Paste the exact phrase, sentence, or block of copy to evaluate here. [CONTEXT] Where this copy appears and what state triggered it. Example: "Post-set coaching note after a user hit all 5 target reps on Squat, third week at 80 kg." Or: "Readiness note shown on HomeView when readiness score is 42 out of 100." [DATA AVAILABLE] What user data was available to the system when this phrase was generated. Example: "Exercise name, weight, reps logged, whether target reps were hit, last 3 sessions for this exercise." Or: "Only profile data — first session, no prior history." Output Format1. VOICE COMPLIANCE SCORE — Rate 1 (fails) to 5 (passes) on each dimension: · Directness: declarative statements, no hedging, no softening · Specificity: references actual data from [CONTEXT] or [DATA AVAILABLE] · Non-cheerleading: no hype words, no empty affirmations · Non-clinical: no medical or data-report language · Active voice: subject acts, not passive constructions 2. VIOLATIONS FOUND — Quote the exact text from the snippet that fails, and name the rule it violates. If no violations, write "None found." 3. REWRITE SUGGESTION — A revised version that passes all 5 dimensions. Must use the same data context. Must not introduce new information not present in [DATA AVAILABLE]. 4. APPROVAL RECOMMENDATION — One of: APPROVE — phrase passes all 5 dimensions, ready for phrase bank REVISE — phrase has 1–2 fixable issues; rewrite provided REJECT — phrase fails core voice rules; structural problem, not a line edit 5. SIMULATION TO RUN — Which CSIM scenario (01–15) would most usefully validate this phrase with a real persona, and why. Be specific about what the persona run would reveal that the LLM evaluation cannot.
Expected Output Boxes
Voice Compliance Score
Five dimensions rated 1–5. A phrase ready for the bank scores 4 or 5 on all dimensions. A score of 3 or below on any single dimension triggers REVISE or REJECT.
Violations Found
Exact quoted text with the named rule. "None found" is a valid result. Vague violations ("this could be more specific") without a quote do not count.
Rewrite Suggestion
Revised version using only the data in [DATA AVAILABLE]. Must not invent new information. Mark DRAFT until reviewed by the content team lead.
Approval & CSIM Routing
APPROVE / REVISE / REJECT with the specific CSIM scenario recommended for live validation. Cardio or readiness copy should always route to CSIM-04 or CSIM-10.
