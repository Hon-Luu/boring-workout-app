Section 1 — Must-Pass Growth Criteria
★ 1. App Store Optimization — ASO 8 tests ▼
GR-001
App Name
App name is ≤30 characters, unique in the App Store search for "workout tracker" and "strength training", and communicates the product's purpose or identity.
Pass: Name search on App Store returns H.O.N. in top 5 for "habit workout tracker"; character count ≤30.
GR-002
Subtitle
Subtitle is ≤30 characters, contains at least 2 high-traffic keywords not in the app name ("strength log", "fitness coach", "gym tracker"), and communicates the core value in plain language.
Pass: Subtitle tested against keyword tool; 2+ keywords with >50k monthly searches included.
GR-003
Description — Above the Fold
First 3 lines (visible without "More") communicate the core value proposition without requiring the user to tap. No feature list in the first paragraph — the opening is about the user's problem, not the app's features.
Pass: First 3 lines read aloud in <10 seconds; convey why H.O.N. exists and who it's for.
GR-004
Screenshots
3 screenshots minimum, optimized for iPhone 15 Pro Max (6.7") dimensions. Screenshot 1 shows the most distinctive UI moment (not the home screen). Each screenshot has a headline that communicates a benefit, not a feature name.
Pass: Screenshots reviewed; headline copy passes the voice test (direct, specific, not generic).
GR-005
Preview Video
15–30 second preview video shows real app usage (not animations of UI elements), starts with the most compelling moment in the first 3 seconds, and has no voiceover or text that says "Download now."
Pass: Video reviewed; first 3 seconds show a high-engagement UI state (active workout or PR moment); no download CTA.
GR-006
Keywords Field
100 characters used fully; no duplicate keywords from name or subtitle; includes long-tail terms (e.g., "progressive overload tracker", "workout journal", "gym log for iPhone").
Pass: Keyword field is exactly 100 characters; no single word appears more than once across name + subtitle + keywords.
GR-007
Rating Strategy
Review request is triggered at the right moment (after a PR, after session 10, after a milestone) — never after an error, never mid-workout.
Pass: Review request trigger defined in code; not triggered during a workout or after any error state.
GR-008
App Store Category
Primary category is Health & Fitness; secondary category chosen to maximize visibility in a less competitive category (e.g., Sports).
Pass: Both categories confirmed; app appears in browse under both.
➔ 2. Onboarding Conversion 8 tests ▼
GR-009
Time to Value
User reaches their first meaningful interaction (logging a set OR receiving a workout plan) within 3 minutes of install.
Pass: Timed with a fresh install; ≤3 minutes to first value moment.
GR-010
Onboarding Completion Rate
≥85% of users who start onboarding complete it (reach the home screen). Biggest drop-off step identified and addressed.
Pass: TestFlight data shows ≥85% completion; biggest drop-off step identified and addressed.
GR-011
Permission Request Timing
HealthKit permission requested after the user has seen value (after first session or home screen render), not on first launch.
Pass: Fresh install; HealthKit permission not shown until after reaching home screen.
GR-012
Value Preview Before Permissions
Before asking for any permission, the app shows the user something compelling: their first workout template, the readiness system explained, or a sample coaching note.
Pass: User sees at least one compelling UI moment before any permission dialog appears.
GR-013
Skip Option on Optional Steps
Every onboarding step that asks for optional data (name, body weight, equipment) has a visible skip. Skipping does not break any downstream feature.
Pass: All optional steps have a skip affordance; skipping does not break any downstream feature.
GR-014
Progress Indicator
Onboarding shows how many steps remain. User is never surprised by how long onboarding is.
Pass: Progress dots or step counter visible throughout onboarding.
GR-015
Onboarding → First Session Same-Day Rate
% of users who complete onboarding AND log a session on the same day. Target: ≥60%.
Pass: ≥60% same-day session rate in TestFlight.
GR-016
Re-Onboarding for Returning Users
A user who deletes and reinstalls the app reaches their first session without re-doing the full onboarding (if their data is backed up or if they've been recognized).
Pass: Reinstall flow handles returning user gracefully; not forced through onboarding again if prior data exists.
● 3. Lifecycle & Retention Mechanisms 8 tests ▼
GR-017
Day 1 Notification
User receives a motivating (not generic) push notification within 20–24 hours of install if they have not returned. Notification is exercise-specific based on their onboarding goal.
Pass: Notification copy reviewed; references the user's goal or equipment (not "Don't forget to work out!").
GR-018
Gap Detection
If a user has not logged a session in 5 days, the app prepares a comeback experience (not necessarily a notification — could be HomeView state on next open).
Pass: 5-day gap triggers a specific HomeView state distinct from the normal state.
GR-019
Streak System Is Informational, Not Anxiety-Inducing
The streak counter shows the current streak without threatening language if it's about to break. No "Your streak ends tonight!" Breaking a streak is presented neutrally on next open.
Pass: Streak display reviewed; no urgency language. Breaking a streak is presented neutrally on next open.
GR-020
Win Moment After First PR
After a user's first PR, the app has a moment worth sharing (screenshot-worthy UI, sharable stat card, or at minimum a memorable visual).
Pass: Trigger a PR; UI produces a shareable or memorable moment. User would consider sharing it.
GR-021
D7 Re-Engagement Notification
For users who haven't opened the app in 7 days, a push notification is sent. Notification references a specific fact about their training (last exercise, days since last session, upcoming goal milestone).
Pass: Notification copy references at least one fact from the user's data. Not generic.
GR-022
D30 Habit Reinforcement
Users who reach 30 days of any activity receive a message that reinforces their identity as an athlete and explains what their consistency has produced.
Pass: D30 message exists; it references their actual session count or progress, not a generic "30 days!" message.
GR-023
In-App Notification Opt-Out
User can opt out of push notifications from within the app (not just iOS Settings).
Pass: Settings screen has a notifications preference that reduces frequency or disables non-critical notifications.
GR-024
Lapse Recovery Funnel (30+ Days)
For users who have been inactive for 30+ days, a specific win-back sequence exists (could be a notification or a HomeView comeback state). Warm, not guilt-inducing.
Pass: 30+ day inactive user opens app; experience is distinct from 7-day gap experience. Warm, not guilt-inducing.
↗ 4. Word of Mouth & Referral 6 tests ▼
GR-025
Shareable PR Moment
When a user hits a personal record, the visual state is distinctive enough that they'd want to share it. A screenshot of the PR moment should communicate the achievement without requiring explanation.
Pass: PR UI screenshot shown to someone unfamiliar with the app; they understand it's a personal record without explanation.
GR-026
Shareable Milestone
Session 25, 50, 100 milestone moments are visually distinct and shareable. The achievement is stated clearly: "Session 100. 100 times you chose to show up."
Pass: Milestone screenshot is self-explanatory to an outsider.
GR-027
No Forced Sharing
The app never requires sharing to unlock a feature or advance. Sharing is always optional.
Pass: Zero share gates in the app. All features accessible without sharing.
GR-028
Natural Recommendation Moment
The app asks "Do you know anyone who'd benefit from this?" or similar only at a high-satisfaction moment (post-PR, post-milestone, post D30). Never during onboarding, never after a mediocre session.
Pass: Recommendation prompt timing reviewed; only appears at high-satisfaction states.
GR-029
App Store Review Request Timing
The review request fires at one of: post-PR, post-milestone (session 10+), or post D7 retention. Never mid-workout, never after an error.
Pass: Trigger conditions reviewed in code; confirmed to match the specified moments.
GR-030
"Told by a Friend" Onboarding Path
If a user says they heard about the app from a friend during onboarding, the experience acknowledges it (even just a single line: "Welcome — glad your friend sent you.").
Pass: Referral source question exists in onboarding; positive response triggers a specific acknowledgement.
◇ 5. Monetization Readiness 6 tests ▼
GR-031
Value Before Paywall
User experiences the full core loop (log a workout, receive coaching, see progress) at least once before encountering any paywall.

Paywall Framing
If a paywall exists, it explains what the user gets (not what they're missing). "Unlock full analytics, advanced coaching, and unlimited trainer plans" not "You've hit your free limit."

Free Tier Is Genuinely Useful
Users who never pay can still get real value from the app indefinitely (not crippled). Define the free tier features; confirm they constitute a complete experience for a casual user.
Pass: Free tier features defined; confirm they constitute a complete experience for a casual user.

Pass: UI layout presents the intended tier as visually primary.

Pass: Settings screen has a visible subscription management link within 2 taps of the home screen.
GR-036
Trial Conversion Notification Timing
If a free trial exists, the conversion notification fires 24 hours before trial expiry (not 1 hour — that feels aggressive).
Pass: Trial expiry notification timing confirmed at T-24h.

Section 3 — Definition of Done for Growth

H.O.N. is growth-ready when the App Store listing can be understood and evaluated by a stranger in 8 seconds, when a new user reaches their first logged set within 3 minutes of install, when at least 85% of users who start onboarding finish it, and when the first person to hit a PR would consider sharing a screenshot without being prompted. Done means all 46 growth criteria are green, the day-1 notification references something specific about who the user is and what they want — not a generic push-to-open, the streak system shows progress without threatening the user when life gets in the way, and lapsed users who return after 30 days are welcomed back with warmth rather than confronted with failure. Done for growth means the app earns its own word-of-mouth because the moments inside it are worth talking about — not because we built a referral mechanic, but because we built something people actually want to tell their friends about. Done does not mean viral. It means every growth mechanism in this document is grounded in real user behavior, tested against real cohort personas, and honest about what the product does and who it's for.
