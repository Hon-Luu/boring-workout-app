Ship Blockers — Do Not Launch Until Fixed

BLOCKER 1 — Data Corruption: dropWeight bug

Set records can store a dropWeight value wildly inconsistent with the main set weight (e.g., 106 kg on a 12.5 kg dumbbell exercise), causing e1RM to display as "Elite" when the user is Intermediate. This is a data integrity issue that directly misleads users about their performance.

Fix One-time migration in SeedStore sanitizes any dropWeight > 5× mainWeight. Clears cached PRs. Runs once via UserDefaults flag. ✓ Implemented this session.
BLOCKER 2 — Empty Onboarding (5/12 personas affected)

First launch shows an empty home screen with no guidance. Sam, Danny, Kyle, Wei, and Alex all experience confusion or abandonment in Phase 1. No welcome, no sample workout, no tooltip on the "+" button.

Fix Add a first-launch welcome sheet with 3 steps: (1) "Tap + to start your first workout" (2) "We'll track your progress automatically" (3) "You own your data — no account needed"
BLOCKER 3 — Classification Explanation Missing (8/12 personas)

The strength classification badge ("Beginner / Intermediate / Advanced / Elite") is displayed with no explanation of what it means, how it's calculated, or what a user should do to advance. Priya, Marcus, Tom, Rachel, Serena, James, Chloe, and Kyle all express confusion or distrust about the label.

Fix Tap badge → explanation sheet: "Based on your e1RM relative to your bodyweight using Epley formula. Thresholds: Beginner <0.5×, Intermediate 0.5–1.0×, Advanced 1.0–1.5×, Elite >1.5×."
BLOCKER 4 — Drop Set Invisible in History (resolved this session)

Before this session's fix, the read-only workout history and the edit sheet both failed to show drop sets. Users who logged drop sets had no way to verify, review, or correct them.

Fix HistoryView.swift updated: read-only now shows "↳ Drop" row; EditableSetRow now has bindings for dropWeight/dropReps/isDropCompleted. ✓ Implemented this session.
BLOCKER 5 — No Success Feedback After Save (7/12 personas)

After editing and saving a workout, saving bodyweight, or finishing a session, there is no confirmation toast or visual feedback. Users must re-navigate to verify the save. Several personas (Marcus, Priya, Rachel) report uncertainty about whether their edits were persisted.

Fix Add a 1.5s banner toast after any save action: "Saved ✓" in sage color, bottom of screen, auto-dismiss.
BLOCKER 6 — Weight Stepper in Dead Zone (one-handed usage)

The weight stepper is positioned top-center of the Set Logger, unreachable with one hand on a 6.7" iPhone. 9/12 personas use the app in a gym context (one hand free, other hand holds phone). Kyle and Tom explicitly note awkwardness adjusting weight between sets.

Fix Move weight stepper to mid-screen or add ± quick-tap buttons at bottom alongside the reps stepper. The "Log Set" CTA should be the southernmost element.

Cross-Persona Patterns (A–H)

PATTERN A — The Repetitive Exercise Problem

9/12 personas log the same 4–6 exercises every session. The exercise picker treats every session as novel. No recents = 3+ unnecessary taps per session.

PATTERN B — The Trust Gap

Users distrust numbers they can't explain. Showing "Elite" without a formula tap-target, or "208 kg e1RM" without a sanity check, breaks trust faster than a bad UX.

PATTERN C — The One-Hand Problem

Gym context = one hand on phone, one hand between sets or holding equipment. Any control above the mid-screen line loses 9/12 users in their primary context.

PATTERN D — The Silent Save

Every save action (finish workout, edit history, update bodyweight) is silent. Users in 7/12 journeys re-navigate to confirm. A 1.5s toast would eliminate this entirely.

PATTERN E — The Metric Literacy Gap

"e1RM," "tonnage," "BW-relative" are invisible to 7/12 personas. The analytics engine is excellent but the UI assumes domain expertise the users don't have.

PATTERN F — The Empty State Drop-Off

Sam, Danny, and Alex all hit empty states with no forward framing. "No data" is a dead end. "Your first chart appears after 2 sessions" is a runway.

PATTERN G — The Comeback Moment

After a gap (injury, travel, life), 5/12 personas return to the app feeling shame about the break. The app is silent. A "Welcome back — here's where you left off" moment would rebuild habit.

PATTERN H — The Drop Set Black Box

Drop sets exist in the data model and engine but are invisible in 3 places: history read view, edit sheet, and onboarding. Advanced users (Marcus, James, Tom) rely on them — and until this session, had no way to review or correct them.
