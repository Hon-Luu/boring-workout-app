# User Research

## Primary User Profile (Hon)

- "Falls and gets back up" lifter — habit-focused, not performance-obsessed
- Huberman Protocol listener — values science-backed reasoning, likes the "why"
- Tech + math background — comfortable with advanced metrics in Expert mode
- Wants beauty + depth — not a minimal app, but never cluttered
- Training mix: strength primary, occasional cardio, general activity (hiking, walks)
- Keeps terminal open; works in long sessions with Claude Code

## H.O.N. Core Design Principles

1. **Never punish gaps** — the return is celebrated, not the streak
2. **Numbers serve the habit** — metrics are instruments, not ends
3. **Be honest when data is insufficient** — no false confidence scores
4. **Match the user's voice** — direct, dry, no corporate warmth
5. **Science earns its place** — justify complexity, don't just expose it

## Jobs To Be Done

| When I... | I want to... | So I can... |
|-----------|-------------|-------------|
| Open the app before training | Know if today is a good day to push | Train smarter, not just harder |
| Finish a session | Feel like it mattered | Keep coming back |
| Return after 2+ weeks away | Come back without shame | Restart without starting over |
| Am on a 4-week roll | See the pattern I've built | Believe it's real |
| Hit a plateau | Understand if it's rest, load, or something else | Make a decision, not a guess |

## Validated Assumptions

- User responds well to direct, non-cheerleader tone (HON message library)
- Expert mode (INOL/PSI/e1RM) is desired — but in-context explanation is missing (V7)
- Tappable heatmap cells are high-value — confirmed by user reaction to Pass 4 implementation
