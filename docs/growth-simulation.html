H.O.N.

Growth Simulation Protocol

15 growth scenarios · Channel priority matrix · Observation template · LLM growth pre-triage · Last updated: 2026-05-30
Section 1 — How to Run
Growth simulations measure how well H.O.N. acquires, converts, and retains users through its natural product mechanics and store presence — not paid advertising. Each scenario places a tester in the shoes of a potential or existing user at a specific growth-critical moment: discovery, first impression, conversion decision, referral impulse, or re-engagement. The observer records what drives or blocks the desired behavior. Results feed directly into ASO copy, notification strategy, lifecycle triggers, and referral mechanics. High-priority runs (marked H in the matrix) should be completed before App Store submission.
Section 2 — 15 Growth Scenarios
GSIM-01
App Store First Impression — 8-Second Test
5 min Trigger: App Store listing page ASO Conversion High Priority
What We're Observing
In 8 seconds, does an unfamiliar person understand what H.O.N. is and want to download it?
Priority Cohort
U-02 U-06 U-13 U-17
Steps
  1. Open the App Store listing. Show it to a persona unfamiliar with H.O.N. for exactly 8 seconds, then close it.
  2. Ask: "What is this app?"
  3. Ask: "Who is it for?"
  4. Ask: "Would you download it? Why or why not?"
  5. Show it again for 30 seconds. Ask: "What would make you more likely to download?"
  6. Note: did they read the title, subtitle, screenshots, or description first?
Pass / Fail Criteria
Pass
Person correctly describes H.O.N.'s core purpose and target user. At least half say they'd consider downloading.
Fail
Person is confused about what the app is, or says "another workout app."
GSIM-02
Onboarding Speed Run
8 min Trigger: Fresh install Time to First Value High Priority
What We're Observing
How long from install to first logged set? Where does momentum slow?
Priority Cohort
U-02 U-13 U-06 U-17
Steps
  1. Fresh install. Start timer.
  2. Persona completes onboarding and logs their first set — with no coaching from observer.
  3. Stop timer when first set is logged.
  4. Ask: "Was there any step that felt unnecessary or confusing?"
  5. Ask: "At any point did you want to give up?"
  6. Note exact point where persona hesitated >5 seconds.
Pass / Fail Criteria
Pass
First set logged in ≤3 minutes. No step caused a hesitation >10 seconds.
Fail
>3 minutes, or persona expresses desire to quit at any point.
GSIM-03
HealthKit Permission Timing Reception
5 min Trigger: First launch — HealthKit permission Permission Opt-in Rate
What We're Observing
Does the timing of the HealthKit request feel earned or premature? Would the persona grant or deny?
Priority Cohort
U-02 U-06 U-13 U-15
Steps
  1. Fresh install. Navigate to the point where HealthKit permission appears.
  2. Ask persona: "Before you answer — does this feel like the right moment to ask?"
  3. Ask: "Do you understand what data this app will access?"
  4. Ask: "Would you grant or deny, and why?"
  5. Note: if permission was asked before any value was shown, record this as a timing failure.
Pass / Fail Criteria
Pass
Persona grants permission. They feel the timing was logical (value was shown first).
Fail
Persona denies because "the app hasn't earned it yet" or they don't understand what data is accessed.
GSIM-04
Day-1 Re-engagement Notification
5 min Trigger: 20 hours post-install, no return D1 Re-engagement Rate High Priority
What We're Observing
Would the persona tap the notification? Does it feel relevant to them specifically?
Priority Cohort
U-02 U-13 U-17 U-15
Steps
  1. Read the notification text aloud (or show it as a mock lock screen banner).
  2. Ask: "Would you tap this? Why or why not?"
  3. Ask: "Does this feel like it knows something about you, or is it generic?"
  4. Ask: "Would you turn off notifications for this app after seeing this?"
  5. Note: does the notification reference the user's goal or equipment?
Pass / Fail Criteria
Pass
Persona says they'd tap. Notification references their specific goal or context.
Fail
Persona says "I'd ignore this" or "I'd turn off notifications." Notification is generic ("Time to work out!").
GSIM-05
Shareable PR Moment
8 min Trigger: Set that beats all-time e1RM Word-of-Mouth Trigger High Priority
What We're Observing
Is the PR visual state compelling enough that the persona would share it? Does the screenshot communicate the achievement to someone who doesn't use H.O.N.?
Priority Cohort
U-01 U-20 U-09 U-05
Steps
  1. Seed exercise history. Persona logs the PR-triggering set.
  2. Observe facial/body reaction to the PR treatment.
  3. Ask: "Would you take a screenshot of this and share it?"
  4. If yes: "Who would you send it to? What would you say?"
  5. Show the PR screenshot to a third party (not the persona). Ask the third party: "What happened here?"
  6. Note: does the third party understand it's a personal record without explanation?
Pass / Fail Criteria
Pass
Persona would share. Third party correctly identifies it as a PR without explanation.
Fail
Persona says "share what?" UI doesn't communicate the achievement clearly enough to share.

Session 10 Milestone Reception

Is the milestone moment compelling enough to make a persona want to tell someone about the app?
Priority Cohort

GSIM-07
Word-of-Mouth Prompt Reception
5 min Trigger: Post-PR or milestone referral prompt Referral Mechanics
What We're Observing
Does the referral prompt feel earned and genuine, or like a product demand? Who specifically does the persona think of?
Priority Cohort
U-07 U-01 U-09 U-13
Steps

  2. Ask: "Who specifically comes to mind?"
  3. Ask: "Would you actually reach out to them within 24 hours?"

Pass

Fail
No one comes to mind. Prompt feels like a product demand. Description is generic ("a workout app").
GSIM-08
App Store Review Request Reception
5 min Trigger: Post-PR or post-session-10 review request Review Conversion Rate
What We're Observing
Does the timing feel earned? Does the persona tap to leave a review?
Priority Cohort
U-05 U-07 U-01 U-20
Steps
  1. Trigger the review request at the specified moment.
  2. Observe whether persona taps "Rate" or dismisses.
  3. If they tap — note that they would actually leave a review.
  4. If they dismiss — ask: "What would have made you more likely to rate?"
  5. Ask: "Does the timing of this request feel right, or did it interrupt something?"
  6. Note: was the moment genuinely high-satisfaction?
Pass / Fail Criteria
Pass
Persona taps to leave a review OR says they would have if they had a moment. Timing described as "natural" or "good timing."
Fail
Persona dismisses. Describes the request as "premature" or "disruptive."
GSIM-09
Referral Source Onboarding — Friend Referral Path
8 min Trigger: "Heard from a friend" onboarding source Referral Conversion
What We're Observing
Does the app acknowledge the referral source in a way that feels personal and warm?
Priority Cohort
U-02 U-13 U-17 U-06
Steps
  1. Fresh install. During onboarding, select "A friend told me" as the source.
  2. Observe what happens — does the app acknowledge it?
  3. Ask: "Did the app acknowledge that a friend sent you?"
  4. Ask: "How did that make you feel about the app before you even started?"
  5. If no acknowledgement: ask "Would a personal welcome have changed your first impression?"

Pass / Fail Criteria
Pass
App acknowledges referral source with a specific line. Persona feels welcomed.
Fail
App treats the persona the same as any new user. No acknowledgement of referral source.
GSIM-10
7-Day Re-engagement Notification
5 min Trigger: 7-day inactive user, data-specific notification D7 Re-engagement High Priority
What We're Observing
Does a data-specific notification outperform a generic one? Would the persona open the app?
Priority Cohort
U-07 U-08 U-15 U-10
Steps
  1. Read two notifications to persona: (a) "Haven't trained in a while. Come back." (b) "[Their exercise] is ready for its next attempt."
  2. Ask: "Which would you be more likely to tap?"
  3. Ask: "Does either of these make you want to open the app right now?"
  4. Ask: "What notification would you actually want to receive after 7 days away?"
  5. Note: persona's ideal notification content vs. what we have.
Pass / Fail Criteria
Pass
Persona prefers the data-specific notification. They'd open the app from it.
Fail
Persona prefers neither. "I'd turn off notifications for this app."
GSIM-11
Free Trial Expiry Notification — 24h Before
5 min Trigger: Notification 24h before trial expiry Trial-to-Paid Conversion
What We're Observing
Does 24 hours feel like enough time to decide? Is the notification framing positive (what you get) or negative (what you'll lose)?
Priority Cohort
U-02 U-17 U-06 U-15
Steps
  1. Read the trial expiry notification.
  2. Ask: "Does this give you enough time to decide?"
  3. Ask: "Does this make you feel like you're about to lose something, or like you're being given a choice?"
  4. Ask: "What would you do right now if you received this?"
  5. Note: FOMO language vs. positive value framing.
Pass / Fail Criteria
Pass
Persona feels they have enough time. Notification is positively framed. Majority would open the app to decide.
Fail
Persona says "24 hours isn't enough" or "this feels like pressure."
GSIM-12
35-Day Lapse Win-Back — Self-Motivated Return
10 min Trigger: 35-day inactive, self-motivated open Comeback Experience Quality
What We're Observing
Does the app's comeback state create enough pull to re-engage a user who came back on their own?
Priority Cohort
U-07 U-10 U-08 U-15
Steps
  1. Seed a 35-day gap (last session 35 days ago).
  2. Open app. Observe HomeView comeback state.
  3. Read the comeback message.
  4. Ask: "Does this make you want to log a workout today?"
  5. Ask: "Does this feel warm, or does it remind you of your absence?"
  6. Ask: "What would you need to see to decide to stay?"
  7. Note: does the comeback state make re-entry feel easy or effortful?
Pass / Fail Criteria
Pass
Persona says yes to working out today. Comeback state described as warm. Re-entry feels low-friction.
Fail
Persona says "maybe later" or feels the gap is highlighted in a negative way.
GSIM-13
App Store Search Discovery Competition
8 min Trigger: Search results for "workout tracker" Search Result Competitiveness High Priority
What We're Observing
In a real search context, would a persona choose to tap on H.O.N. over competitors?
Priority Cohort
U-02 U-17 U-13 U-06
Steps
  1. Show search results page (mock or real) including H.O.N. and 4 competitors. Give persona 10 seconds to look at it.
  2. Ask: "Which would you tap and why?"
  3. If they don't choose H.O.N. first: "What about H.O.N. made it less attractive?"
  4. Ask: "What does H.O.N.'s icon communicate to you?"
  5. Ask: "What does the subtitle tell you that the others don't?"
  6. Note: icon, subtitle, and first screenshot are the primary competitive signals.
Pass / Fail Criteria
Pass
H.O.N. is in the persona's consideration set. They'd tap it. Icon and subtitle are competitive.
Fail
Persona consistently chooses a competitor. H.O.N.'s listing doesn't stand out in search.
GSIM-14
D30 Habit Reinforcement Moment
8 min Trigger: 30-day D30 reinforcement message Long-Term Retention Trigger
What We're Observing
Does the D30 message make the persona feel like a committed athlete, reinforcing the habit identity that keeps them coming back?
Priority Cohort
U-07 U-02 U-05 U-13
Steps
  1. Seed 30 days of activity. Show or read the D30 reinforcement message.
  2. Ask: "How does this make you feel about your training?"

Fail

GSIM-15
Free Tier Longevity — 60-Day No-Payment Experience
15 min Trigger: 60 days seeded usage, free tier only Free Tier Genuine Value High Priority
What We're Observing
After 60 days without paying, does the free tier still feel genuinely useful? Does it create pull toward upgrading (not pressure)?
Priority Cohort
U-17 U-06 U-15 U-02
Steps
  1. Seed 60 days of free-tier usage. Open app as that user.
  2. Navigate all 5 tabs. Ask: "Does this still feel like a complete experience?"
  3. Ask: "Is there anything you wish you had access to that you don't?"
  4. Ask: "Do you feel like the app is deliberately limiting you, or are you genuinely happy with what you have?"
  5. Ask: "If an upgrade option appeared right now, would you consider it? What would it need to offer?"
  6. Note: deliberately crippled vs. naturally incomplete free tier.
Pass / Fail Criteria
Pass
Free tier still feels genuinely useful at 60 days. User sees value in upgrading but doesn't feel forced.
Fail
User describes the free tier as "deliberately crippled" or says "I feel like I'm being pushed to pay."

Section 5 — LLM Growth Pre-Triage Prompt
Growth Pre-Triage — Paste into any LLM before running a growth simulation
Provide the three inputs below. The model will return a structured growth signal assessment that tells you which GSIM scenario to run first and whether any dark-pattern risk exists before you invest simulation time.
Input 1 — Feature or Moment
The specific feature, UI state, or product moment being evaluated (e.g. "PR celebration screen," "D7 re-engagement notification," "session 10 milestone splash").
Input 2 — User State
Where the user is in their lifecycle: new / D1 / D7 / D30 / lapsed / returning. Include any relevant context (first workout, trial expiring, long gap, etc.).
Input 3 — Growth Metric to Move
The specific metric this moment should improve: D1 re-engagement rate / D7 retention / trial-to-paid CVR / App Store review volume / referral rate / DAU / organic installs.
Expected Output
Word-of-Mouth Potential
1–5 score. 5 = high shareability; 1 = internal loop only. Explain which element drives or limits organic spread.
Notification Opportunity
Is this a trigger point for a personalized notification? If yes: what data should the notification reference to feel specific rather than generic?
ASO Keyword Angle
What user intent or search phrase does this feature address? Recommend an ASO keyword or metadata angle if applicable.
Lifecycle Trigger Fit
Which lifecycle stage does this best serve — acquisition, activation, retention, referral, or revenue? Is the current placement aligned with that stage?
Paywall Leverage
If applicable: does this moment create natural pull toward an upgrade? Or does it risk feeling like a gate? Note if paywall is not applicable for the current context.
Recommended GSIM
Which GSIM-## scenario should be run first based on these inputs, and with which cohort (U-##)? Brief rationale.
Dark Pattern Risk
1–5 score where 5 = high risk. Flag if the mechanic uses urgency, FOMO, guilt, or social pressure in a way that could erode trust or generate negative reviews. If risk ≥ 3: name the specific pattern and recommend a reframe.
Usage note: This triage is pre-simulation, not a replacement for it. Use the output to prioritize which GSIM to run and to identify potential framing issues before they show up in persona observations. A low dark-pattern risk score does not mean the feature is effective — only that it's not manipulative. Run the recommended GSIM to validate actual behavior.
